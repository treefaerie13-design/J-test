# 日本語復習 — 日语复习测验系统

一个纯静态的日语主动回忆测验网页：答题（选择/填空）、按语法点和语义场统计弱点、随手收集查词内容进收件箱。数据存在你自己的私有 GitHub 仓库里，手机电脑打开同一个网址即可通用；不配置 GitHub 时也能作为纯本地应用使用（文件导入导出）。

## 快速开始

### 方式一：纯本地（零配置）

1. 双击 `index.html`。
2. 「数据」页 → 「载入题库文件」→ 选择你的 `questions.json`。
3. 答题记录自动存在浏览器 localStorage；**定期用「导出进度」下载 JSON 备份**（清浏览器缓存会丢 localStorage，文件才是可靠存储）。

### 方式二：GitHub 同步（推荐，多设备）

1. **建两个仓库**：
   - 应用仓库（公开）：放本项目文件，Settings → Pages → 从 main 分支部署，得到网址 `https://<用户名>.github.io/<仓库名>/`。
   - 数据仓库（**私有**，建议名 `jp-quiz-data`）：放 `questions.json`、`inbox.json`、`progress/` 目录（模板见配套的 jp-quiz-data 仓库）。
2. **申请 Token**：GitHub → Settings → Developer settings → Personal access tokens → **Fine-grained tokens** → Generate new token：
   - Repository access：**只选数据仓库**这一个。
   - Permissions → Repository permissions → **Contents: Read and write**。其余全部不给。
   - 有效期按需（到期重新生成粘贴即可）。
3. 打开网页 → 「数据」页 → 填入用户名、数据仓库名、token、本设备名（如 `desktop`、`phone`）→ 保存并连接。
4. 每台设备各配一次（设备名不要重复，答题记录按设备分文件存，避免冲突）。

**手机**：浏览器打开同一网址，可「添加到主屏幕」当 App 用。

## 安全须知

- Token 只授予了数据仓库的读写权限，泄露的最坏后果是这个仓库的数据被读改——里面只有你的题库和答题记录。
- Token 存在浏览器 localStorage，只发送给 `api.github.com`。不要在公用电脑上保存配置（用完点「清除配置」）。

## 日常使用

- **答题**：可按类型/标签/语法点筛选，支持「只刷错题」。键盘：数字键选项、回车提交/下一题、`0` 我不会、`Esc` 跳过。答错的题之后会更频繁出现（简化版间隔重复）。
  - **我不会**：记为答错并直接展示答案解析——想不起来就诚实点这个，比瞎蒙对统计更有价值，这题之后会优先重出。
  - **跳过此题**：不留任何记录直接换题（适合暂时不想练的题）。
- **收集**：在 Claude 等任何地方查完词，复制粘贴进收集框保存。之后在电脑上让 Claude Code 批量消化成题目（见数据仓库 README）。
- **收藏**：答完一道题点「☆ 收藏」把重要知识点收进来；「收藏」页可浏览（直接看题+答案+解析）、新建/重命名收藏夹并把题整理进去，每个收藏夹可单独「练这组」（收藏夹专练时忽略其他筛选，点「结束」退出）。收藏数据存在数据仓库 `favorites.json`，多设备同步。
- **弱点报告**：按正确率从低到高排序，最虚的语法点排最前；可导出 Markdown 贴进 Obsidian。
- **坏题**：答题时发现题目有问题（答案歧义、干扰项也对），点「这题有问题」，它会移出抽题池且不再污染统计，之后在电脑端批量修。

## 加题

三种方式，格式模板见数据仓库的 `questions.json`：

1. 手编 JSON 直接提交到数据仓库。
2. 电脑上用 Claude Code 技能 `/jp`：查词的同时自动生成题目入库。
3. 收件箱攒原料 → Claude Code `/digest-inbox` 批量出题。
4. 被标记的坏题用 `/fix-flagged` 批量修。

## 技术说明

- 无构建步骤、无外部依赖、无 CDN/字体请求；同步只访问 `api.github.com` 读写你自己的仓库。
- 部署 = 把这三个文件 push 到开了 Pages 的仓库。
- 简化版间隔重复的取舍与未来接入真正 SRS 的位置，见 `app.js` 中「抽题」一节的注释。
